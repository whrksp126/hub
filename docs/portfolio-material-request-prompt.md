# 포트폴리오 재료 요청 — 프로젝트 담당 agent용 마스터 프롬프트

> **용도**: hub.ghmate.com 포트폴리오에 각 프로젝트(ghc·openday·skyswarm·codingpt 등)를 오더앤고/헤이보카 수준으로 만들기 위해, **해당 프로젝트 담당 agent에게** 문서·실제 스크린샷·시연 영상·기술 딥다이브를 **한 번에** 요청하는 프롬프트.
>
> **사용법 (두 가지)**
> 1. 아래 프롬프트에서 `{{프로젝트명}}`을 실제 프로젝트명으로 바꿔 담당 agent에게 붙여넣기.
> 2. 또는 담당 agent에게: *"이 문서(`/Users/whrksp126/other/project/hub/docs/portfolio-material-request-prompt.md`)를 참고해서 `{{프로젝트명}}` 포트폴리오 재료를 만들어줘"* 라고 경로만 전달.
>
> **확립된 원칙(오더앤고·헤이보카에서 배운 것)**: 팀/역할 추정 금지 · dev/test 로그인으로 실제 화면 캡처 · **데모 계정을 풍성하게 시딩**해 화면이 꽉 차게 · 실제 mp4/스크린샷 산출(스토리보드 아님) · 분할 멀티화면+스텝자막+가상커서 영상 규격 · 딥다이브는 아래 마크다운 형식(파서 호환) · 보안 배제.

## 프롬프트 본문 (여기부터 복사)

안녕! 나는 whrksp126의 포트폴리오(hub.ghmate.com)에 **`{{프로젝트명}}` 프로젝트 상세 + 기술 딥다이브**를 만드는 에이전트야. 이미 완성한 **오더앤고·헤이보카와 같은 수준**으로, **문서·실제 스크린샷·시연 영상·딥다이브를 한 번에** 만들어줘. (발행은 내가 함 — 너는 정확·풍부한 재료와 실제 자산만.)

### 0. 먼저 확인 (추정 금지)
- **팀/역할**: 혼자인지, 팀이면 **본인 담당 범위를 시기별로** 정확히. (git 커밋 기여자와 어긋나지 않게 정직하게 — "혼자 개발"로 단정하지 말 것.)
- **로그인 벽 통과 경로**: 실제 화면 캡처/녹화를 위해 **dev/test 로그인 경로**(예: `/auth/dev-login`, dev-only 로그인 버튼, 시드 계정)를 찾아 사용. 없으면 로컬에서 만들어서라도 실제 로그인 화면을 캡처.

### 1. 데모 계정 데이터 세팅 (★ 화면이 꽉 차 보이게 — 가장 중요)
스크린샷/영상이 **비어 보이지 않게**, 테스트 계정 상태를 **다양하고 풍성하게** 미리 채워줘:
- 목록/컬렉션은 **여러 개**, 상태값은 **한쪽에 쏠리지 않고 골고루 분포**(진행률·단계·카테고리 등이 다양하게 보이게).
- 핵심 시각 요소(차트·통계·배지·진행바·상태 뱃지 등)가 **의미 있게 그려질 만큼**의 데이터.
- 알림/경고·업적·보상·이력 등이 **활성 상태**로 보이게(비어있는 empty state 회피).
- 개인정보 없는 **데모 데이터**로만. 화면에 뜨는 이름/이메일은 데모용.
> 세팅한 내용을 매니페스트에 적어줘(재현 가능하게).

### 2. 문서 재료 (텍스트) — "쉽고·빠르고·간편"
1. **한 줄 핵심** — 누구(고객)에게 어떤 가치인지 한 문장.
2. **개요** — 기존의 불편/비싼 방식 → 이 서비스가 어떻게 간단히 해결하는지. 짧은 산문 + **가치 불릿 3~4개**.
3. **지표 3개** — 숫자보다 **의미 중심**(value + label).
4. **핵심 기능** — 사용자가 체감하는 **모든 기능 면을 빠짐없이**, "제목 — 한 줄 설명".
5. **역할** — 0에서 답한 팀/담당을 **시기별로** 정확히.
6. **기술 스택 + 선택 이유**.
7. **확장 방향** — 미착수는 "지향/구상"으로 정직하게.
8. **전체 구조 mermaid flowchart**(다크테마, 소스 텍스트).

### 3. 스크린샷 (실제 캡처) — 데모 계정 로그인 상태로
- 위 풍성한 상태에서 **핵심 화면 전부** 캡처. UI가 있으면 **레티나(DSF2)**, 모바일은 세로(예: 402×874@2x), 데스크톱은 1920×1080↑.
- **라이트/다크** 등 테마가 있으면 페어로.
- CLI/백엔드 등 화면이 없는 프로젝트면 **터미널 실행·출력·결과 캡처**로 대체.

### 4. 시연 영상 (실제 녹화) — 오더앤고/헤이보카 스타일
- **연동/인과가 보이는 건 분할 멀티화면**(A에서 하면 → B에 반영), 상단 **제목바** + 좌/우 라벨. 단일 흐름은 세로 단일 화면.
- **스펙**: 데스크톱 분할 **1920×1080** / 모바일 **1080×2340**, **DSF2**, mp4(**H.264·yuv420p·crf20·faststart**), **무음**, **가상 마우스 커서 + 클릭 리플**, 동작 사이 0.5~1초 여유, 8~18초.
- **앱 진입/대시보드/스플래시 인트로 금지** — 바로 기능 화면에서 시작.
- **하단 중앙 스텝 자막**(반투명 알약 + 흰색 44px, 페이드, 스텝당 ≥2초, 전 영상 동일 디자인) — 각 단계 "지금 뭐 하는지" 한 줄.
- 패널/모달은 **오버레이+슬라이드**(기존 콘텐츠 밀지 말 것). 애니메이션은 **선명히 보이게** 페이싱.
- 각 영상 **포스터 대표컷**(클라이맥스 프레임, 자막 포함) 동봉.
- 실기기 녹화가 화질상 유리하면 그쪽으로, 헤드리스면 업스케일 여부를 매니페스트에 명시.

### 5. 기술 딥다이브 3편 이상 (문제 1개 = 글 1개)
**보안(필수)**: 시크릿·API키·비밀번호·토큰·내부호스트/IP/SSH·실제 카드/승인번호·개인정보·악용 가능한 취약점 상세 **금지**. 코드는 핵심 로직만 남긴 **일반화 예시**(경로·크리덴셜·식별자 마스킹). 영상/캡처에 민감정보 보이면 블러.

각 글: **배경·문제 → 제약 → 설계·해결(단계별) → 구현(일반화 코드) → 결과 → 배운 점 → 다이어그램 + 시연 영상/스크린샷**.
